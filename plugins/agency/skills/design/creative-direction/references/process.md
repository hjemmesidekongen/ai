# Creative Direction Generator — Process Reference

## Purpose

This document contains the full generation logic, output schema, and
derivation rules for creative-direction.yml. The SKILL.md file references
this for execution details.

## Prerequisites

Before running this skill:
- brand-loader must have completed (checkpoint: brand_reference_loaded)
- `.ai/projects/{project}/brand/brand-summary.yml` must exist with colors,
  typography, and spacing sections
- Read `plugins/agency/skills/design/frontend-design/SKILL.md` and its
  reference files to internalize Layer 1 constraints

## Input: brand-summary.yml

Expected structure from brand-loader output:

```yaml
brand_name: "..."
personality: [list of personality traits]
audience: "target audience description"
tone: [list of tone descriptors]
product_type: "SaaS | e-commerce | portfolio | editorial | ..."
colors:
  primary: { hex, scale: [10-step] }
  secondary: { hex, scale: [10-step] }
  accent: { hex, scale: [10-step] }
  neutrals: { scale: [10-step] }
typography:
  heading_family: "..."
  body_family: "..."
  scale: "major-third | perfect-fourth | ..."
spacing:
  base: "..."
  scale: [...]
```

If personality, audience, tone, or product_type are missing, infer from
available brand signals (colors, typography choices, brand name). Document
inferences in findings.md.

## Output Schema: creative-direction.yml

Every field is required. The file must be valid YAML.

```yaml
# creative-direction.yml
# Generated by creative-direction skill — do not edit manually

project: "{project}"
generated_from: "brand-summary.yml"
layer: 3

identity:
  metaphor: "one-line metaphor that captures the brand's visual essence"
  # Example: "A precision instrument — Swiss watchmaking meets digital craft"

feel:
  primary: "dominant emotional descriptor"
  secondary: "supporting emotional descriptor"
  tertiary: "subtle undertone"
  # Derived from personality + audience intersection

motion_philosophy:
  approach: "minimal | expressive | cinematic | playful"
  enter_style: "how elements enter the viewport"
  exit_style: "how elements leave (if applicable)"
  duration_bias: "fast (150-250ms) | moderate (250-400ms) | slow (400-700ms)"
  easing_preference: "sharp | smooth | bouncy | spring"
  scroll_animations: true | false
  # Must respect Layer 1 motion rules from frontend-design

spatial_philosophy:
  density: "airy | balanced | compact"
  whitespace_role: "breathing room | luxury signifier | content separator"
  section_rhythm: "uniform | alternating | progressive"
  max_content_width: "narrow (680px) | standard (1024px) | wide (1280px) | full"
  # Driven by product_type and audience expectations

texture:
  surface: "flat | subtle-depth | layered | glassmorphic"
  borders: "none | hairline | defined | bold"
  shadows: "none | subtle | medium | dramatic"
  backgrounds: "solid | gradient | pattern | image-heavy"
  grain_or_noise: true | false

interaction_weight:
  hover_intensity: "none | subtle | moderate | dramatic"
  click_feedback: "instant | animated | ripple"
  form_style: "minimal | outlined | filled"
  button_personality: "sharp | rounded | pill | ghost-primary"
  cursor_effects: true | false

color_strategy:
  dominant: "which color role dominates — primary | neutral | accent"
  contrast_level: "high | moderate | low"
  gradient_use: "none | subtle-accents | hero-gradients | pervasive"
  dark_mode_approach: "invert | dim | separate-palette | not-supported"
  accent_usage: "CTAs-only | highlights-and-CTAs | decorative"
  # Must pass Layer 1 WCAG contrast requirements

typography_personality:
  heading_weight: "light | regular | bold | black"
  heading_case: "sentence | title | uppercase | lowercase"
  body_line_height: "tight (1.4) | comfortable (1.6) | spacious (1.8)"
  pull_quote_style: "italic | large | bordered | highlighted"
  monospace_use: "none | code-blocks | decorative | data"
  # Must respect Layer 1 typographic scale rules

hero_approach:
  style: "image-led | text-led | split | video | animated | minimal"
  height: "viewport | 70vh | auto"
  overlay: "none | gradient | color-wash | blur"
  cta_position: "center | left-aligned | bottom | floating"
  headline_size: "display-1 | display-2 | h1"

scroll_behavior:
  page_feel: "static | smooth-scroll | parallax-subtle | parallax-heavy"
  reveal_pattern: "instant | fade-up | stagger | slide-in"
  sticky_elements: "none | nav-only | nav-and-sidebar | progressive"
  infinite_scroll: true | false
  progress_indicator: "none | bar | dots | percentage"

anti_patterns:
  - "description of a project-specific thing to avoid"
  # These are ADDITIONAL to Layer 1 anti-slop rules
  # Derived from brand personality — e.g., a luxury brand avoids
  # cartoon illustrations; a developer tool avoids stock photography
```

## Derivation Logic

### Step 1: Classify the Brand Archetype

Map personality traits to one of these archetypes:

| Archetype | Signals | Direction Bias |
|-----------|---------|----------------|
| Premium/Luxury | elegant, refined, exclusive | Airy, slow motion, dramatic shadows |
| Technical/Precise | accurate, reliable, systematic | Compact, fast motion, flat surfaces |
| Playful/Creative | fun, bold, expressive | Expressive motion, bouncy easing, color-heavy |
| Warm/Approachable | friendly, caring, inclusive | Balanced density, moderate motion, rounded shapes |
| Minimal/Modern | clean, simple, focused | Airy, minimal motion, flat, high whitespace |
| Bold/Disruptive | edgy, innovative, challenger | Cinematic motion, dramatic contrasts, sharp edges |

Most brands blend 2 archetypes. Use the primary as the foundation, secondary
as the modifier.

### Step 2: Derive Each Field

For each of the 11 sections in creative-direction.yml:

1. Start with the archetype defaults
2. Adjust based on specific personality traits
3. Adjust based on audience expectations (enterprise audiences expect
   restraint; consumer audiences tolerate more expression)
4. Adjust based on product_type (SaaS needs functional density; portfolio
   needs visual breathing room)
5. Cross-check against Layer 1 rules — if a derived value contradicts
   frontend-design constraints, Layer 1 wins

### Step 3: Generate Anti-Patterns

Anti-patterns are project-specific. Derive them from the brand identity:

- What would feel "off-brand" given the personality?
- What design trends conflict with the audience?
- What does the competitive landscape overuse?

Generate 3-7 anti-patterns. These supplement (not replace) the Layer 1
anti-slop rules from frontend-design.

### Step 4: Internal Consistency Check

Verify the generated direction is internally consistent:

- motion_philosophy.duration_bias aligns with feel (fast for energetic,
  slow for premium)
- spatial_philosophy.density matches texture.surface (airy + layered is
  inconsistent)
- interaction_weight matches motion_philosophy (heavy interactions need
  motion budget)
- color_strategy.contrast_level meets Layer 1 accessibility requirements
- anti_patterns don't contradict positive direction fields

If inconsistencies exist, resolve in favor of the brand's primary archetype.

## Validation Checklist

Before marking checkpoint complete:

- [ ] creative-direction.yml exists at the correct path
- [ ] All 11 top-level sections present with no empty values
- [ ] identity.metaphor is a single compelling line
- [ ] feel has exactly 3 descriptors (primary, secondary, tertiary)
- [ ] motion_philosophy respects Layer 1 motion constraints
- [ ] color_strategy passes Layer 1 WCAG contrast requirements
- [ ] typography_personality respects Layer 1 scale rules
- [ ] anti_patterns has 3-7 entries, all project-specific
- [ ] No internal contradictions between sections
- [ ] File is valid YAML (parseable without errors)

## Example Derivation

**Input brand signals:** personality=[precise, innovative, trustworthy],
audience="enterprise DevOps teams", tone=[direct, technical, confident],
product_type="SaaS"

**Archetype classification:** Technical/Precise (primary) + Bold/Disruptive
(secondary, from "innovative")

**Resulting direction highlights:**
- identity.metaphor: "Mission control — calm authority with real-time precision"
- feel: confident / systematic / subtly bold
- motion: minimal approach, fast duration, sharp easing
- spatial: balanced density, standard width (SaaS needs functional space)
- texture: subtle-depth, hairline borders, subtle shadows
- interaction: moderate hover, instant feedback, outlined forms
- color_strategy: neutral-dominant, high contrast, CTAs-only accents
- anti_patterns: ["Stock photography of people pointing at screens",
  "Playful illustrations or mascots", "Rounded pill buttons (too casual)",
  "Gradient backgrounds (undermines precision feel)"]
