# Execution Guide: SEO Plugin

## Reality Check

Building seo-plugin is a 15-step process. Each step builds one skill or
command, with a checkpoint at the end. You'll need approximately 15
sessions in Claude Code, doing one skill per session with `/compact` between.

**Rules:**
- One skill per Claude Code session
- `/compact` between sessions
- CLAUDE.md auto-updates after every step
- Git commit after every step
- Specs live in docs/ — Claude reads files, don't paste content
- If a step fails its checkpoint, fix it before moving on

## Prerequisites

Before starting, these spec documents must exist in docs/:
- `seo-plugin-implementation-plan.md` — generated by plugin-spec-generator
- `seo-plugin-addendum.md` — generated by plugin-spec-generator

### Step 3: Skill — project-interview

> **What:** Gathers project context including website URL, industry, goals, target audience, and current SEO status through an interactive interview
> **Interactive:** yes
> **Depends on:** none

**Prompt:**

---

Read `docs/seo-plugin-implementation-plan.md` (the "Skill: project-interview" section under Skills).
Also read the brand-reference.yml schema in `docs/ecosystem-strategy.md` Section 6 to know what brand data is available.
Read `docs/seo-plugin-addendum.md` (the "Domain Knowledge" section) for SEO context.

Create `plugins/seo-plugin/skills/project-interview/SKILL.md`

This skill:
1. Load brand context if available — call brand-context-loader to read `identity.mission`, `identity.values`, `identity.positioning`, and `audience.segments` from brand-reference.yml
2. Initialize seo-strategy.yml at `~/.claude/seo/[project-name]/` with the `meta` section containing plugin_name ("seo-plugin"), project_name, created_at (ISO 8601), version ("1.0"), generated_by ("seo-plugin v1.0.0"), and brand_name
3. Ask the user for their website URL — validate it matches a URL pattern (starts with http:// or https://)
4. Ask about their industry and business model — offer examples: "SaaS, E-commerce, Agency, B2B Services, Media, Education." If brand identity is loaded, suggest based on identity.positioning
5. Ask about 2-3 primary SEO goals — present goal-metric pairs as examples: "Increase organic traffic → monthly_organic_sessions", "Improve keyword rankings → average_position", "Grow backlink profile → referring_domains", "Increase conversions from organic → organic_conversion_rate". Each goal needs a target value and timeframe
6. Ask about target audience segments — if brand audience personas are loaded, present them and ask the user to confirm or adjust for SEO context. Each segment needs segment name, search_behavior (unaware/problem-aware/solution-aware/product-aware), and preferred_content format
7. Assess current SEO status — ask about estimated domain authority (0-100), number of indexed pages, any currently ranking keywords, and known SEO issues
8. Compile all answers into the `project_context` section of seo-strategy.yml with fields: website_url, industry, goals (array), target_audience (array), current_status (object with domain_authority, indexed_pages, top_keywords, known_issues)
9. Write the completed meta and project_context sections to seo-strategy.yml
10. Run checkpoint validation

Brand data sections needed: identity (mission, values, positioning), audience (segments with goals, pain_points)

Output: writes `meta` and `project_context` sections of `seo-strategy.yml`

Checkpoint type: data_validation
Required checks:
- project_context.website_url is a valid URL format (starts with http:// or https://)
- project_context.industry is non-empty string
- project_context.goals has at least 2 entries, each with goal (string), metric (string), target (string), and timeframe (string) fields
- project_context.target_audience has at least 2 segments, each with segment (string), search_behavior (enum), and preferred_content (string)
- project_context.current_status has domain_authority (number), indexed_pages (number), and known_issues (array) fields
- meta section has all required fields: plugin_name, project_name, created_at, version, generated_by

Then update CLAUDE.md: check off this step in the Progress section and set "Next step" to the following step. Commit everything.

### Step 4: Skill — keyword-research

> **What:** Conducts keyword research identifying primary, secondary, and long-tail keywords with search volume estimates, difficulty scores, and search intent classification
> **Interactive:** yes
> **Depends on:** project-interview

**Prompt:**

---

Read `docs/seo-plugin-implementation-plan.md` (the "Skill: keyword-research" section under Skills).
Also read the brand-reference.yml schema in `docs/ecosystem-strategy.md` Section 6 to know what brand data is available.
Read `docs/seo-plugin-addendum.md` (the "Keyword Research Methodology" section) for domain knowledge on search intent classification and keyword categorization.

Create `plugins/seo-plugin/skills/keyword-research/SKILL.md`

This skill:
1. Read seo-strategy.yml section `project_context` to get industry, goals, target_audience, and current_status.top_keywords
2. Load brand voice and content sections from brand-reference.yml via brand-context-loader — extract voice.vocabulary (brand-specific terms), voice.attributes, and content.seo_guidelines
3. Generate a seed keyword list of at least 10 keywords based on: the project's industry, each SEO goal's metric area, target audience segment names and search behaviors, and brand vocabulary terms
4. Present the seed keywords to the user in a numbered list — ask them to confirm (keep), remove, or add keywords. One interaction round: show list, get feedback, finalize
5. For each confirmed seed keyword, generate 2-3 secondary keyword variations (modifiers, related terms) and 3-5 long-tail keyword variations (question-based, specific phrases)
6. Estimate search volume (monthly searches) and difficulty (0-100 scale) for each keyword using domain knowledge of the industry — base estimates on industry benchmarks from the addendum
7. Classify search intent for every keyword as one of: informational (seeking knowledge), navigational (seeking a specific site), commercial (comparing options), transactional (ready to buy/sign up)
8. Present the full keyword list organized by category (primary, secondary, long-tail) — ask the user to set priority (high/medium/low) for primary keywords
9. Validate: at least 5 primary keywords, at least 10 secondary keywords, at least 15 long-tail keywords — each with search_volume (number), difficulty (number 0-100), search_intent (enum), and priority (enum, primary only)
10. Check for duplicate keywords across all three categories — remove any duplicates
11. Write the `keywords` section to seo-strategy.yml with primary, secondary, and long_tail arrays
12. Run checkpoint validation

Brand data sections needed: voice (vocabulary, attributes), content (seo_guidelines, writing_rules)

Output: writes `keywords` section of `seo-strategy.yml`

Checkpoint type: data_validation
Required checks:
- At least 5 primary keywords, each with search_volume (number), difficulty (number 0-100), search_intent (enum: informational|navigational|commercial|transactional), and priority (enum: high|medium|low)
- At least 10 secondary keywords with search_volume, difficulty, and search_intent fields
- At least 15 long-tail keywords with search_volume, difficulty, and search_intent fields
- Every keyword has search_intent set to one of: informational, navigational, commercial, transactional
- No duplicate keywords across primary, secondary, and long_tail arrays

Then update CLAUDE.md: check off this step in the Progress section and set "Next step" to the following step. Commit everything.

### Step 5: Skill — competitor-analysis

> **What:** Analyzes top competitors for target keywords, identifying their strengths, weaknesses, content gaps, and backlink profiles
> **Interactive:** no
> **Depends on:** project-interview

**Prompt:**

---

Read `docs/seo-plugin-implementation-plan.md` (the "Skill: competitor-analysis" section under Skills).
Read `docs/seo-plugin-addendum.md` (the "Competitive Analysis for SEO" section) for domain knowledge on competitor analysis methodology.

Create `plugins/seo-plugin/skills/competitor-analysis/SKILL.md`

This skill:
1. Read seo-strategy.yml section `project_context` to get industry, website_url, and current_status
2. Identify 3-5 likely competitors based on: the project's industry, business model, and geographic market. Use domain knowledge to suggest realistic competitors for the given industry
3. For each competitor, estimate domain authority (0-100) based on industry knowledge — established players in the industry typically range 50-80, newer competitors 20-40
4. Identify each competitor's top 5-10 ranking keywords with estimated SERP positions and traffic — base on common industry terms that competitors in this space would rank for
5. Analyze content strengths for each competitor — identify 2-4 specific strengths (e.g., "Comprehensive long-form guides averaging 3000+ words", "Active blog with 2 posts per week", "Strong video content library")
6. Analyze content weaknesses for each competitor — identify 2-4 specific weaknesses (e.g., "No FAQ or structured data", "Thin product comparison pages", "Blog hasn't been updated in 6 months")
7. Estimate backlink profile for each competitor — total backlinks, referring domains, and 2-3 notable linking domains
8. Identify content gaps by cross-referencing competitor weaknesses — find at least 5 keywords or topics where competitors have missing, thin, or outdated content
9. Score each content gap with an opportunity_score (0-100) based on: estimated search volume (higher = better), competitor weakness severity (missing > thin > outdated), and alignment with the project's goals
10. Write the `competitors` section to seo-strategy.yml with analyzed (array of competitor profiles) and content_gaps (array of gap opportunities)
11. Run checkpoint validation

Output: writes `competitors` section of `seo-strategy.yml`

Checkpoint type: data_validation
Required checks:
- At least 3 competitors analyzed, each with domain (string), domain_authority (number 0-100), and top_keywords (array with at least 3 entries)
- Each competitor has at least 2 content_strengths and 2 content_weaknesses entries
- Each competitor has backlink_profile with total_backlinks (number), referring_domains (number), and top_sources (array)
- content_gaps array has at least 5 entries, each with keyword (string), gap_type (enum: missing|thin|outdated), opportunity_score (number 0-100), and recommended_action (string)

Then update CLAUDE.md: check off this step in the Progress section and set "Next step" to the following step. Commit everything.

### Step 6: Skill — technical-seo

> **What:** Produces a technical SEO checklist covering site speed, mobile-friendliness, crawlability, indexation, structured data, and Core Web Vitals targets
> **Interactive:** no
> **Depends on:** project-interview

**Prompt:**

---

Read `docs/seo-plugin-implementation-plan.md` (the "Skill: technical-seo" section under Skills).
Read `docs/seo-plugin-addendum.md` (the "Technical SEO Fundamentals" section) for domain knowledge on Core Web Vitals, crawlability, and mobile-first indexing.

Create `plugins/seo-plugin/skills/technical-seo/SKILL.md`

This skill:
1. Read seo-strategy.yml section `project_context` to get website_url and current_status (particularly known_issues)
2. Define Core Web Vitals targets: LCP target ≤ 2.5 seconds, FID target ≤ 100 milliseconds, CLS target ≤ 0.1 score — include unit and description for each metric
3. Generate the technical checklist with at least 10 items across these categories:
   - crawlability: XML sitemap, robots.txt, canonical URLs, 404 handling, breadcrumbs, URL structure
   - security: SSL/HTTPS, security headers
   - mobile: viewport meta tag, touch target sizes (48x48px min), no horizontal scroll, font size (16px base)
   - performance: page load time, image optimization, lazy loading, CDN usage, caching
   - accessibility: image alt tags, heading hierarchy, ARIA labels
   - rich_results: structured data (Organization, Article, BreadcrumbList), Google Rich Results Test
4. Assign priority to each item: critical (blocks ranking), high (significant impact), medium (moderate impact), low (nice to have)
5. Set all items to status "pending" — these are recommendations, not assessments
6. Cross-reference checklist items with project_context.current_status.known_issues — if a known issue matches a checklist item, add a note referencing it
7. Define mobile requirements: viewport meta tag set, touch targets 48x48px minimum, no horizontal scrolling, font size 16px base
8. Ensure at least 4 categories are represented with at least 2 items each
9. Write the `technical` section to seo-strategy.yml with core_web_vitals (object), checklist (array), and mobile_requirements (array)
10. Run checkpoint validation

Output: writes `technical` section of `seo-strategy.yml`

Checkpoint type: data_validation
Required checks:
- core_web_vitals has targets for lcp (target ≤ 2.5, unit "seconds"), fid (target ≤ 100, unit "milliseconds"), and cls (target ≤ 0.1, unit "score")
- checklist has at least 10 items across at least 4 categories
- Each checklist item has item (string), priority (enum: critical|high|medium|low), status (enum: pending|in_progress|done|na), and category (string)
- At least 2 items have priority "critical"
- mobile_requirements has at least 4 entries

Then update CLAUDE.md: check off this step in the Progress section and set "Next step" to the following step. Commit everything.

### Step 7: Skill — on-page-optimization

> **What:** Generates on-page SEO rules including title tag patterns, meta description templates, heading structure, internal linking strategy, and schema markup recommendations
> **Interactive:** no
> **Depends on:** project-interview, keyword-research

**Prompt:**

---

Read `docs/seo-plugin-implementation-plan.md` (the "Skill: on-page-optimization" section under Skills).
Read the "Skill: keyword-research" section in the implementation plan to understand the keyword data this skill consumes.
Read `docs/seo-plugin-addendum.md` (the "Technical SEO Fundamentals" section, specifically the structured data paragraph).

Create `plugins/seo-plugin/skills/on-page-optimization/SKILL.md`

This skill:
1. Read seo-strategy.yml section `project_context` to get website_url and industry
2. Read seo-strategy.yml section `keywords` to get primary keywords (for title tag and heading optimization) and secondary keywords (for meta descriptions and subheadings)
3. Generate title tag pattern using the brand name and primary keyword placement — set max_length to 60 characters. Define at least 2 rules: primary keyword within first 3 words, brand name position
4. Generate meta description template with action verb, benefit statement, and CTA structure — set max_length to 155 characters. Define at least 2 rules: include primary keyword naturally, end with call-to-action
5. Define heading hierarchy rules for H1 through H4:
   - H1: one per page, contains primary keyword
   - H2: major sections, include secondary keywords
   - H3: subsections, include long-tail keywords where natural
   - H4: use sparingly for deep nesting
6. Create internal linking strategy with at least 3 rules:
   - Every page links to at least 3 related pages (contextual links in body content)
   - Pillar pages link to all cluster content (maintain topic cluster map)
   - New content links to existing high-authority pages (check existing content for opportunities)
   Each rule has a `rule` (string) and `implementation` (string) field
7. Recommend at least 2 schema markup types based on the project type — for each, list the required_properties array and use_case string. Common types: Organization (homepage), Article (blog), FAQ (help pages), Product (e-commerce), BreadcrumbList (all pages)
8. Write the `on_page` section to seo-strategy.yml with title_tag, meta_description, heading_hierarchy, internal_linking, and schema_markup
9. Run checkpoint validation

Output: writes `on_page` section of `seo-strategy.yml`

Checkpoint type: data_validation
Required checks:
- title_tag has pattern (string), max_length (number ≤ 60), and rules (array with at least 2 entries)
- meta_description has pattern (string), max_length (number ≤ 155), and rules (array with at least 2 entries)
- heading_hierarchy has rules defined for h1, h2, h3, and h4 — each with a rules array
- internal_linking has at least 3 entries, each with rule (string) and implementation (string)
- schema_markup has at least 2 entries, each with type (string), required_properties (array), and use_case (string)

Then update CLAUDE.md: check off this step in the Progress section and set "Next step" to the following step. Commit everything.

### Step 8: Skill — content-strategy

> **What:** Creates a content plan with topic clusters, content calendar framework, content types, and SEO-optimized content guidelines aligned with brand voice
> **Interactive:** no
> **Depends on:** keyword-research, competitor-analysis

**Prompt:**

---

Read `docs/seo-plugin-implementation-plan.md` (the "Skill: content-strategy" section under Skills).
Also read the brand-reference.yml schema in `docs/ecosystem-strategy.md` Section 6 to know what brand data is available.
Read `docs/seo-plugin-addendum.md` (the "Topic Clusters and Content Architecture" section) for domain knowledge on pillar-cluster content models.
Read the "Skill: keyword-research" and "Skill: competitor-analysis" sections in the implementation plan to understand the data this skill consumes.

Create `plugins/seo-plugin/skills/content-strategy/SKILL.md`

This skill:
1. Read seo-strategy.yml section `keywords` to get primary keywords (for pillar topics), secondary keywords (for cluster content), and long-tail keywords (for supporting pages)
2. Read seo-strategy.yml section `competitors.content_gaps` to identify content opportunities where competitors are weak
3. Load brand voice, audience, and content sections from brand-reference.yml via brand-context-loader — extract voice.attributes (for content tone), audience.segments (for targeting), and content.content_types (for format guidelines)
4. Group primary keywords into at least 3 topic clusters — each cluster has one pillar page targeting a high-volume primary keyword. Assign secondary and long-tail keywords to supporting pages within each cluster
5. For each pillar page, define: title (string), target_keyword (from primary keywords), word_count (3000-6000), content_type ("ultimate_guide")
6. For each topic cluster, define at least 3 supporting pages with: title (string), target_keyword (from secondary or long-tail keywords), search_intent (from keyword data), word_count (1500-3000), content_type (enum: how_to|listicle|case_study|comparison|tutorial), priority (high|medium|low)
7. Define at least 3 content types with guidelines: type name, word_count_range (string), structure description, and required seo_elements (array of strings)
8. Create a 3-month content calendar with publishing cadence — define schedule as array of months, each with content entries listing title, target week, and status ("planned")
9. Incorporate competitor content gaps as priority content opportunities — reference the gap's keyword and recommended_action
10. Align content tone with brand voice attributes — reference voice spectrum and channel variations from brand-reference.yml
11. Write the `content_plan` section to seo-strategy.yml with topic_clusters (array), content_types (array), and calendar (object with cadence, months_planned, schedule)
12. Run checkpoint validation

Brand data sections needed: voice (attributes, channel_variations), audience (segments, personas), content (content_types, seo_guidelines)

Output: writes `content_plan` section of `seo-strategy.yml`

Checkpoint type: data_validation
Required checks:
- At least 3 topic clusters defined
- Each topic cluster has a pillar page with title (string), target_keyword (string), word_count (number 3000-6000), and content_type (string)
- Each topic cluster has at least 3 supporting pages with title, target_keyword, search_intent, word_count, content_type, and priority fields
- content_types array has at least 3 entries, each with type (string), word_count_range (string), structure (string), and seo_elements (array)
- calendar.months_planned is at least 3
- calendar.schedule has entries for all planned months with content arrays

### Step 10: Skill — compile-and-export

## Tips

1. **Read the spec, don't memorize it.** Every prompt tells you which file and section to read.
   Claude Code reads the file — you don't need to paste content.

2. **One skill per session.** Context degrades over long sessions. Build one skill,
   commit, `/compact`, start fresh.

3. **Checkpoints are gates.** If a checkpoint fails, the skill is not done.
   Fix the issues before moving to the next step.

4. **Build order matters.** Skills are ordered so each one can use the previous
   skill's output. Don't skip ahead.

5. **Interactive skills need the user.** If a skill is marked interactive,
   the user must be present to answer questions. Plan accordingly.

6. **Brand data is loaded, not asked.** If the plugin uses brand data,
   the brand-context-loader handles finding and loading it.
   Skills declare what sections they need — the loader provides them.

7. **Recovery works.** If a session crashes, check state.yml.
   It records exactly where you left off. Use `/plan:resume` to continue.
