# Implementation Plan v2: Brand Guideline Plugin
## With Cross-Plugin Brand Architecture

---

## The Core Idea

The brand guideline plugin produces **two outputs**:

1. **Brand Manual** (for humans) — A polished, agency-grade document you can hand to a designer, share with a client, or print. Beautiful, readable, comprehensive.

2. **Brand Reference File** (for Claude) — A structured, machine-readable file that lives in a known location. Every other plugin (SEO, social media, website builder, etc.) knows where to find it and pulls what it needs automatically.

This means when you later run `/seo:report`, it doesn't ask "what's your brand voice?" — it already knows. When `/website:build` starts, it already has the color palette, typography, and personas loaded. The brand guideline becomes the **foundation layer** of your entire plugin ecosystem.

---

## Architecture: How Plugins Share Brand Context

```
~/.claude/brands/
├── my-company/
│   ├── brand-reference.yml        ← Machine-readable (all plugins read this)
│   ├── brand-manual.docx          ← Human-readable (the polished document)
│   ├── brand-manual.md            ← Human-readable (markdown version)
│   └── assets/
│       ├── logo-primary.png
│       ├── logo-mark.png
│       └── ...
├── client-acme-corp/
│   ├── brand-reference.yml
│   ├── brand-manual.docx
│   ├── brand-manual.md
│   └── assets/
│       └── ...
└── client-widgets-inc/
    └── ...
```

**Any plugin** can load the active brand context with a shared skill:

```
/seo:report --brand my-company
/social:content-plan --brand client-acme-corp
/website:plan --brand my-company
```

If no `--brand` flag is given, Claude asks which brand to use (or defaults if only one exists).

## The Brand Reference File (brand-reference.yml)

This is the most important file in the entire system. It's what every other plugin reads. Here's the full schema:

```yaml

# Auto-generated by /brand:generate or /brand:analyze
# Other plugins consume this file — do not restructure.
# Last updated: 2026-02-28
# ============================================================

  brand_name: "Company Name"
  tagline: "Short tagline"
  industry: "SaaS / E-commerce / Agency / etc."
  website: "https://example.com"
  generated_date: "2026-02-28"
  version: "1.0"

# --------------------------------------------------
# IDENTITY
# --------------------------------------------------
identity:
  mission: "One sentence mission statement"
  vision: "One sentence vision statement"
  values:
    - name: "Innovation"
      description: "Brief description of what this means for the brand"
    - name: "Transparency"
      description: "..."
  brand_story: |
    2-3 paragraph brand origin story / narrative.
  positioning:
    category: "What category you compete in"
    target: "Who you serve (one line)"
    differentiator: "What makes you different (one line)"
    proof: "Why anyone should believe this"

# --------------------------------------------------
# VISUAL IDENTITY
# --------------------------------------------------
visual:
  logo:
    primary: "assets/logo-primary.png"
    mark: "assets/logo-mark.png"
    variants:
      - name: "Full color"
        file: "assets/logo-full-color.png"
        use_on: "Light backgrounds"
      - name: "White / reversed"
        file: "assets/logo-white.png"
        use_on: "Dark backgrounds, photography"
      - name: "Monochrome"
        file: "assets/logo-mono.png"
        use_on: "Single-color applications"
    clear_space: "Minimum clear space equal to height of logo mark on all sides"
    minimum_size: "24px height for digital, 10mm for print"
    misuse_rules:
      - "Never stretch or distort"
      - "Never change colors outside approved palette"
      - "Never place on busy backgrounds without overlay"
      - "Never add effects (shadows, gradients, outlines)"
  imagery:
    style: "Photography / Illustration / Mixed"
    mood: "Describe the overall mood — e.g., warm, authentic, energetic"
    subjects: "What appears in imagery — e.g., real people, product shots, abstract"
    avoid: "What to never show — e.g., stock-looking poses, dark/moody"
  layout:
    grid: "Describe grid system if applicable"
    spacing_principle: "Generous whitespace / compact / etc."

# --------------------------------------------------

# --------------------------------------------------

      use: "Subsections, card titles"
    - level: "Body"
      size: "16px / 1rem"
      weight: "Regular"
      line_height: "1.6"
      use: "Paragraphs, descriptions"
    - level: "Small / Caption"
      size: "14px / 0.875rem"
      weight: "Regular"
      line_height: "1.4"
      use: "Labels, metadata, footnotes"

# --------------------------------------------------
# COLORS
# --------------------------------------------------
colors:
  primary:
    - name: "Brand Blue"
      hex: "#2563EB"
      rgb: "37, 99, 235"
      use: "Primary CTA, links, key UI elements"
  secondary:
    - name: "Slate"
      hex: "#475569"
      rgb: "71, 85, 105"
      use: "Body text, secondary elements"
  accent:
    - name: "Amber"
      hex: "#F59E0B"
      rgb: "245, 158, 11"

  neutrals:

  semantic:
    success: "#16A34A"

    accent: "Amber (10%)"

# --------------------------------------------------
# TONE OF VOICE
# --------------------------------------------------

  personality:
    - attribute: "Confident"
      not: "Arrogant"
      description: "We state things clearly. We don't hedge or over-qualify."
    - attribute: "Warm"
      not: "Casual"
      description: "We're approachable and human, but still professional."
    - attribute: "Clear"

  messaging:
    tagline: "Short tagline"
    value_propositions:

      - "Second key benefit"
      - "Third key benefit"
    elevator_pitch: |
      2-3 sentence pitch that captures who you are,
      what you do, and why it matters.
    boilerplate: |
      Standard 'about us' paragraph for press releases,
      partnership announcements, etc.
  writing_samples:
    - context: "Homepage hero"
      good: "Ship faster with tools that just work."
      bad: "We provide industry-leading solutions for deployment optimization."
      why: "Direct, active voice, benefit-focused vs. jargon-heavy and vague."
    - context: "Error message"
      good: "That didn't work. Try again, or contact us for help."
      bad: "Error 403: Request forbidden by administrative rules."
      why: "Human, helpful, actionable vs. technical and unhelpful."

# --------------------------------------------------
# AUDIENCE & PERSONAS
# --------------------------------------------------
audience:
  primary_market: "Brief description of primary market"
  personas:
    - name: "Persona Name"
      role: "Job title or life role"
      age_range: "28-40"
      quote: "A sentence that captures their mindset"
      goals:
        - "What they're trying to achieve"
        - "Second goal"

        - "What's blocking them"
        - "Second frustration"
      channels:
        - "Where they spend time online"
        - "Second channel"

      decision_factors:
        - "What influences their buying decision"
        - "Second factor"
    - name: "Second Persona"
      role: "..."
      # ...same structure

# --------------------------------------------------
# CONTENT RULES
# --------------------------------------------------
content:
  dos:
    - "Use active voice"
    - "Lead with benefits, not features"
    - "Use 'you' to address the reader directly"
    - "Keep sentences under 20 words when possible"
  donts:
    - "Don't use jargon without explaining it"
    - "Don't use 'synergy', 'leverage', 'utilize'"
    - "Don't write in passive voice"
    - "Don't make promises we can't keep"
  terminology:
    preferred:
      - term: "customers"
        not: "users, clients"
        context: "External communications"
      - term: "team members"
        not: "employees, staff, resources"
        context: "Internal communications"
    industry_terms:
      - term: "Term"
        definition: "Definition in our context"
  grammar:
    oxford_comma: true
    capitalization: "Sentence case for headings"
    numbers: "Spell out one through nine, numerals for 10+"
    date_format: "Month DD, YYYY"
  content_types:
    - type: "Blog post"
      length: "800-1500 words"
      tone_adjustment: "Slightly more casual, educational"
      structure: "Hook → Problem → Solution → CTA"
    - type: "Email marketing"
      length: "150-300 words"
      tone_adjustment: "Direct, action-oriented"
      structure: "Subject → One key message → CTA"
    - type: "Social media"
      length: "Varies by platform"
      tone_adjustment: "Most casual, conversational"
      structure: "Hook → Value → CTA or question"
    - type: "Product / UI copy"
      length: "As short as possible"
      tone_adjustment: "Functional, clear, no personality at cost of clarity"
      structure: "Action-oriented, verb-first"

# --------------------------------------------------
```

## The Brand Manual (for Humans)

The second output is a polished, designed document that covers the same content but is written for a human audience. It's structured as a proper agency deliverable:

### Document Structure

**1. Cover Page**
Brand name, logo, "Brand Guidelines v1.0", date

**2. Introduction**
Why this document exists, how to use it, who it's for

**3. Brand Foundation** *(from identity section)*
- Mission, vision, values
- Brand story
- Positioning statement

**4. Our Audience** *(from audience section)*
- Market overview
- Persona cards (visually formatted, one per page)
- How we speak to each persona

**5. Visual Identity** *(from visual section)*
- Logo: usage, variants, clear space, minimum size, misuse examples
- Color palette: swatches with codes, usage proportions, accessibility
- Typography: type samples, scale, pairing rationale
- Imagery: style direction, mood, do/don't examples
- Layout: grid, spacing, alignment principles

**6. Voice & Tone** *(from voice section)*
- Personality attributes (the "we are / we are not" framework)
- Tone spectrum with sliders
- Writing samples: good vs. bad with explanations
- Messaging: tagline, value props, elevator pitch, boilerplate

**7. Content Guidelines** *(from content section)*
- Writing dos and don'ts
- Terminology guide
- Grammar and style rules
- Content type specifications

**8. Social Media** *(from social section)*
- Platform-by-platform guide
- Post templates and examples
- Engagement rules

**9. Appendix**
- Asset inventory and file locations
- Quick-reference cheat sheet (one page summary)
- Version history and changelog

### Format Options
- **.docx** — For sharing, printing, client delivery
- **.md** — For version control, easy editing
- **.pdf** — For final locked delivery (generated from .docx)

### The Critical Shared Skill: brand-context-loader

This is the linchpin. Every future plugin includes or references this skill:

```markdown
---
name: brand-context-loader
description: >
  Loads the active brand context from brand-reference.yml.
  Used by all brand-aware plugins (SEO, social media, website, content).
  Fires automatically when any plugin needs brand information.
---

## How to load brand context

1. Check if a --brand flag was provided
2. If not, check ~/.claude/brands/ for available brands
3. If only one brand exists, load it automatically
4. If multiple exist, ask which brand to use
5. Read brand-reference.yml and make all sections available
6. Confirm: "Loaded brand context for [Brand Name] (v1.0, updated 2026-02-28)"
```

## How Future Plugins Connect

Once the brand guideline exists, every other plugin becomes simpler:

```
┌─────────────────────────────────────────────────────┐
│                 ~/.claude/brands/                     │
│              brand-reference.yml                      │
│                  (source of truth)                    │
└──────────────────────┬──────────────────────────────┘
                       │
          brand-context-loader (shared skill)
                       │
        ┌──────────────┼──────────────────┐
        │              │                  │
   ┌────▼────┐   ┌────▼────┐      ┌─────▼──────┐
   │   SEO   │   │ Social  │      │  Website   │
   │ Plugin  │   │ Media   │      │  Builder   │
   │         │   │ Plugin  │      │  Plugin    │
   └─────────┘   └─────────┘      └────────────┘

Each plugin:
1. Calls brand-context-loader
2. Gets the sections it needs
3. Does its specialized work using brand context
4. Never asks you to re-explain your brand
```

Example: You run `/seo:report --brand my-company`
- SEO plugin loads brand-context-loader
- Pulls: audience personas, voice tone, content rules, positioning
- Knows your target keywords should match your audience's language
- Knows your content recommendations should follow your voice guidelines
- Produces SEO report that's already aligned with your brand
